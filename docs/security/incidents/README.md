## Incident Response Process

1. **Discovery**: Identify and assess security vulnerability
2. **Triage**: Evaluate impact and severity
3. **Containment**: Stop ongoing exposure
4. **Remediation**: Fix the vulnerability
5. **Validation**: Verify the fix is effective
6. **Documentation**: Record incident and lessons learned

## Evidence Collection

All incidents must include:
- Incident metadata (ID, severity, timeline)
- Evidence of the vulnerability
- Remediation actions taken
- Validation results
- Lessons learned

See individual incident files for examples.
