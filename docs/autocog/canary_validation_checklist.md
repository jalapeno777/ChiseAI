# CANARY Validation Checklist

> 24-hour CANARY mode monitoring and validation framework.
> Generated by ST-AUTOCOG-A-006b.

## Overview

CANARY is an intermediate mode between SHADOW and FULL, where the autonomous cognition system runs with:

- **Max position fraction**: 1% of portfolio (`CANARY_MAX_POSITION_FRACTION = 0.01`)
- **Divergence threshold**: 0.15 (`DIVERGENCE_THRESHOLD_LOW`)
- **Auto-demotion threshold**: 0.40 (`DRIFT_THRESHOLD_FOR_DEMOTION`)
- **Required consecutive checks for promotion**: 5

## Metrics to Monitor

### Primary Metrics

| Metric                             | Description                                   | Source                  |
| ---------------------------------- | --------------------------------------------- | ----------------------- |
| `divergence_score`                 | Measure of neuro-symbolic divergence          | Redis / Cycle artifacts |
| `consecutive_non_regression_count` | Number of consecutive non-regression checks   | Redis                   |
| `position_fraction`                | Portfolio fraction used (max 1% in CANARY)    | Redis                   |
| `trade_count_today`                | Number of trades executed today               | Redis / Cycle artifacts |
| `error_count`                      | Number of errors in current period            | Redis                   |
| `notification_suppressed`          | Whether notifications are properly suppressed | Redis flag              |

### Secondary Metrics

| Metric                      | Description                              | Source          |
| --------------------------- | ---------------------------------------- | --------------- |
| `win_rate`                  | Percentage of profitable trades          | Cycle artifacts |
| `avg_confidence_divergence` | Average confidence divergence            | Cycle artifacts |
| `prediction_drift`          | Prediction drift from baseline           | Cycle artifacts |
| `experiments_per_cycle`     | Number of experiments run (should be ≤1) | Cycle artifacts |

## Success Criteria

CANARY is considered successful when ALL of the following are true:

### Thresholds

- [ ] **Divergence < 0.15**: `divergence_score < DIVERGENCE_THRESHOLD_LOW`
- [ ] **Win rate > 50%**: `winning_trades / total_trades > 0.50`
- [ ] **No critical errors**: `error_count == 0` or `error_rate < 1%`
- [ ] **Notifications working**: Notification suppression verified as correctly functioning
- [ ] **Position fraction ≤ 1%**: `position_fraction <= CANARY_MAX_POSITION_FRACTION`
- [ ] **Experiments within limits**: `experiments_per_cycle <= 1`

### Consistency

- [ ] **5 consecutive non-regression checks**: `consecutive_non_regression_count >= 5`
- [ ] **No drift detected**: `drift_detected == false` for all recent cycles

## Failure Criteria

CANARY should be considered FAILED when ANY of the following are true:

### Critical Failures (Immediate Demotion)

- [ ] **Divergence > 0.40**: `divergence_score > DRIFT_THRESHOLD_FOR_DEMOTION`
- [ ] **Error rate > 5%**: `error_count / total_requests > 0.05`
- [ ] **3+ consecutive losses**: `consecutive_loss_count >= 3`

### Warning Conditions (Monitor closely)

- [ ] **Divergence 0.15-0.40**: Borderline divergence requiring attention
- [ ] **Win rate < 50%**: Underperforming baseline
- [ ] **Notification suppression failure**: Notifications being sent when they should be suppressed

## Auto-Demotion Triggers

The system will automatically demote from CANARY to SHADOW when:

| Trigger            | Threshold                     | Action                       |
| ------------------ | ----------------------------- | ---------------------------- |
| High divergence    | `divergence_score > 0.40`     | Immediate demotion to SHADOW |
| Error rate spike   | `error_rate > 5%`             | Immediate demotion to SHADOW |
| Consecutive losses | `consecutive_loss_count >= 3` | Demotion after review        |
| Manual override    | Human decision                | Immediate demotion to SHADOW |
| Drift detected     | `drift_detected == true`      | Conditional demotion         |

## Validation Schedule

### Hourly Checks (Automated)

```
0h, 1h, 2h, 3h, ... 23h (24 checks total)
```

Each hourly check validates:

1. Current mode is `CANARY`
2. Divergence score is within thresholds
3. Position fraction is ≤ 1%
4. No critical errors in the last hour
5. Notification suppression is functioning

### Automated Scripts

- **`scripts/autocog_canary_health_check.py`**: On-demand health check (exit 0/1/2)
- **`scripts/autocog_canary_report.py`**: Pass/fail report generator

## Manual Intervention Procedures

### When to Intervene Manually

1. **Divergence trending upward**: If divergence increases for 3+ consecutive checks
2. **Unexpected behavior**: System behaving outside expected parameters
3. **Notification issues**: Notifications not being suppressed when expected
4. **Infrastructure problems**: Redis/Qdrant connectivity issues

### Intervention Steps

1. **Review current status**:

   ```bash
   python3 scripts/autocog_canary_health_check.py
   ```

2. **Generate detailed report**:

   ```bash
   python3 scripts/autocog_canary_report.py
   ```

3. **Check recent cycle artifacts**:

   ```bash
   ls -la _bmad-output/autocog/cycles/
   cat _bmad-output/autocog/cycles/autocog-*.json | jq '.metrics'
   ```

4. **Check Redis state**:

   ```bash
   redis-cli KEYS "bmad:chiseai:autocog:*"
   redis-cli GET "bmad:chiseai:autocog:current_mode"
   ```

5. **Make decision**:
   - **Continue CANARY**: All metrics within acceptable ranges
   - **Pause and investigate**: Warnings present but no critical failures
   - **Demote to SHADOW**: Critical failure detected
   - **Promote to FULL**: All success criteria met for 5+ consecutive checks

## Promotion to FULL Mode

CANARY can be promoted to FULL mode when:

1. All success criteria are met (see above)
2. `consecutive_non_regression_count >= REQUIRED_CONSECUTIVE_CHECKS` (5)
3. `divergence_score < PROMOTE_THRESHOLD` (0.35)
4. No demotions have occurred during the CANARY period
5. Win rate has been stable and > 50%
