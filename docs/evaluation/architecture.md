## Checkpoint Format

The checkpoint file maintains scheduler state across restarts:

```json
{
  "state": "running",
  "last_run_6h": 1741000000.0,
  "last_run_daily": 1740900000.0,
  "last_run_weekly": 1740500000.0,
  "cycle_count": 42,
  "error_count": 0,
  "last_error": null,
  "version": "1.0",
  "timestamp": "2026-03-03T12:00:00Z"
}
```

### Fields

| Field | Type | Description |
|-------|------|-------------|
| `state` | string | Current scheduler state |
| `last_run_6h` | float | Unix timestamp of last 6h cycle |
| `last_run_daily` | float | Unix timestamp of last daily cycle |
| `last_run_weekly` | float | Unix timestamp of last weekly cycle |
| `cycle_count` | int | Total cycles executed |
| `error_count` | int | Total errors encountered |
| `last_error` | string\|null | Last error message |
| `version` | string | Checkpoint format version |
| `timestamp` | string | ISO timestamp of checkpoint |

## Threading Model

The scheduler uses two threads:

1. **Main Thread**: Runs the scheduling loop
   - Checks cycle intervals
   - Executes cycles
   - Updates checkpoint
   - Handles shutdown signals

2. **Health Server Thread**: HTTP server for health checks
   - Runs in daemon mode
   - Responds to /health and /status
   - Stops gracefully on shutdown

# - scripts/evaluation/: Evaluation scripts

## Security Considerations

1. **Non-root user**: Container runs as `scheduler` user
2. **Network isolation**: Only connects to chiseai network
3. **Read-only operations**: Never deletes data
4. **Resource limits**: CPU and memory constraints
5. **Health checks**: Validates container health
6. **Graceful shutdown**: Handles signals properly

## Monitoring

### Metrics

The scheduler exposes the following via health endpoint:

- `state`: Current scheduler state
- `cycle_count`: Total cycles executed
- `error_count`: Total errors encountered
- `last_run_*`: Timestamps of last cycle runs

### Logging

Structured JSON logging to:
- Container stdout (captured by Docker)
- File: `{output_dir}/scheduler.log`

### Alerts

Recommended alerts:
- Health check failing for > 5 minutes
- Error count increasing rapidly
- Cycle not running for > 2x interval

## Failure Modes

| Failure | Handling | Recovery |
|---------|----------|----------|
| Redis unavailable | Log warning, continue | Automatic on reconnect |
| Cycle script fails | Log error, increment error_count | Next cycle attempt |
| Signal received | Graceful shutdown | Manual restart |
| Health check fails | Container marked unhealthy | Docker restart |
| Checkpoint corrupt | Reset to defaults | Manual intervention |
